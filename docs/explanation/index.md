## What Makes Explanation Documentation?

Explanation docs should:
- **Clarify and illuminate** concepts and design decisions
- **Provide context and background** for understanding the system
- **Discuss alternatives** and explain why choices were made
- **Connect ideas** and show how pieces fit together
- **Not instruct** - focus on understanding, not doing

## When to Use Explanation Docs

Use explanation documentation when you want to:
- Understand **why** Alexandria works the way it does
- Learn about the **design philosophy** and principles
- Explore **alternative approaches** that were considered
- Gain **deeper insight** into architectural patterns
- Understand **project evolution** and future direction
